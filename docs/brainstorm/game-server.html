Reconnection Handling

If a player disconnects mid-game, the session is not immediately forfeited. The Game Server buffers events for a configurable window (e.g. 60s). If the player reconnects with their token within that window, they receive a full state snapshot and buffered events. If the window expires, the game is awarded to the opponent and the result is reported to the Platform API.
