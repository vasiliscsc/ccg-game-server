Section 4: Action Processing Pipeline

Full flow from action submission to next action dequeue

Main Pipeline

① SUBMIT ACTION
Player action arrives via WebSocket
System action dequeued from IActionQueue
↓
② PHASE GUARD
Reject if action is invalid in current GamePhase
e.g. PlayCard rejected during Mulligan / PendingChoice
SubmitChoice only valid during PendingChoice
SubmitIntervention only valid during PendingIntervention
↓
③ ACTION VALIDATOR
Turn ownership · Mana affordability
Target validity (alive, not stealthed)
Taunt enforcement (must attack Taunt minions)
Attack eligibility (canAttack, not frozen, attacks remaining)
Board space (max 7 minions)
→ Returns error to client on failure, no state mutation
↓
④ DISPATCH → IActionHandler
Mutates GameState · returns events[]
One handler per action type
All state mutation happens here
↓
⑤ PUBLISH EVENTS → IEventBus
For each event (in return order):
Fire subscribers: current player L→R then opponent L→R
Subscribers enqueue new actions via IActionQueue
↓
⑥ AURA RECALCULATION
Run all IAura.Calculate(state)
Update auraAttackBonus / auraHealthBonus on all minions
Minions newly at ≤0 maxHealth marked for death
↓
⑦ DEATH RESOLUTION (wave loop)
See previous diagram for full algorithm
Aura recalculated after each action in Phase 2
Loop until no deaths remain
↓
⑧ WIN CONDITION CHECK
If any hero health ≤ 0:
Both ≤ 0 → Draw · One ≤ 0 → other player wins
Publish GameEndedEvent · phase = Ended
↓
⑨ DEQUEUE NEXT ACTION
Pull next from IActionQueue → back to ①
If queue empty: await next player action

Turn Lifecycle

EndTurnAction handler (active player)
1. Freeze any minions that attacked this turn (if applicable)
2. Publish TurnEndedEvent
3. Fire End-of-Turn triggers (current player L→R, opponent L→R)
4. Swap active player
New turn init (system actions enqueued)
1. Unfreeze minions frozen last turn
2. Increment maxMana (cap 10), restore mana to maxMana
3. Reset canAttack / attacksUsedThisTurn on all minions
4. Reset heroPowerUsedThisTurn, cardsPlayedThisTurn
5. DrawCardAction enqueued
6. Publish TurnStartedEvent
7. Fire Start-of-Turn triggers (new active player L→R, opponent L→R)
8. Start turn timer

PendingChoice Interruption

When an effect calls StartChoiceAction:
• Phase → PendingChoice
• Remaining effect actions stored as continuation in PendingChoice.context
• Pipeline pauses — no further actions processed until choice resolves
• SubmitChoiceAction arrives → fill in selected target/option
• Continuation actions re-enqueued → Phase → InProgress

PendingIntervention Interruption

When engine emits StartInterventionAction:
• Phase → PendingIntervention
• heldAction stored in state — not yet processed
• Timer starts (timeoutSeconds)
• Only SubmitInterventionAction accepted from responding player
• On response or timeout:
  – Card played: process intervention card first, then heldAction
  – Skip/timeout: process heldAction directly
• Phase → InProgress

Aura Recalculation Triggers

Recalculated after any action that changes:
• Board composition (summon, death, bounce, transform)
• Minion stats (buff, silence)
• Keywords added/removed
• After each Phase 2 deathrattle action
• After Phase 3 reborn summons
Key invariant
Aura bonuses are never in enchantments. They are zeroed and fully rewritten on every recalculation. A minion reaching ≤0 maxHealth from aura loss is treated identically to damage death.
