Key Design Rules

Server is authoritative
Client sends intent (action). Server validates, applies, and emits what actually happened.
Client renders events only
No client-side game state mutations. The client is a view layer over the event stream.
Match notify via polling or WS
Client polls Platform API for match status, or holds a lightweight WebSocket connection while in queue.
Spectators are first-class
Spectators connect to Game Server and receive the same event stream, scoped to public information only.
